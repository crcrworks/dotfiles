# Plan: Disable Sisyphus and Use OpenCode Default Plan/Build Agents

## Overview
Remove sisyphus and planner-sisyphus models from oh-my-opencode configuration and use opencode's default plan and build agents instead.

## Files to Modify
- `/Users/crcr/.local/share/chezmoi/dot_config/opencode/oh-my-opencode.json`

## Background

According to oh-my-opencode documentation:
- **Sisyphus agent** is a "batteries-included" agent that replaces opencode's default plan agent
- The `sisyphus_agent` configuration section has two key options:
  - `disabled`: Enable/disable the Sisyphus agent
  - `replace_plan`: When true, demotes default plan agent to subagent mode
- Currently your config has:
  - `sisyphus_agent.disabled: false` (enabled)
  - `sisyphus_agent.replace_plan: false` (not replacing plan agent)
  - Agent configurations for "Sisyphus" and "orchestrator-sisyphus"

## Implementation Steps

### Step 1: Disable Sisyphus agent
Modify `/Users/crcr/.local/share/chezmoi/dot_config/opencode/oh-my-opencode.json`:

Change line 7 from:
```json
"disabled": false,
```
to:
```json
"disabled": true,
```

This will disable the Sisyphus agent system-wide.

### Step 2: Remove Sisyphus agent configurations (optional but recommended)
Remove the following sections from the `agents` object:
- Lines 11-22: "Sisyphus" agent configuration
- Lines 59-61: "orchestrator-sisyphus" agent configuration

These lines define the models and permissions for Sisyphus agents, which are no longer needed when Sisyphus is disabled.

### Step 3: Keep other agent configurations
Preserve all other agent configurations (oracle, explore, librarian, build, plan, etc.) as they provide useful specialized functionality independent of Sisyphus.

## Alternative Approach (Simpler)

If you prefer a simpler approach that doesn't remove the agent configurations (allowing for future re-enablement):

Add to the root of `/Users/crcr/.local/share/chezmoi/dot_config/opencode/oh-my-opencode.json`:
```json
"disabled_agents": [
  "Sisyphus",
  "orchestrator-sisyphus"
],
```

This approach:
- Keeps Sisyphus configurations intact
- Allows potential re-enablement via `@` syntax in the future
- More explicit about which agents are disabled

## Recommended Approach

**Use Step 3 (Alternative Approach)** as it provides better flexibility and clarity while achieving the same goal.

## Verification

After making changes, verify by:

1. **Check configuration validity**:
   ```bash
   cat ~/.config/opencode/oh-my-opencode.json | jq .
   ```
   (or simply read the file to ensure JSON is valid)

2. **Test opencode behavior**:
   - Start a new opencode session
   - Use plan mode to verify opencode's default plan agent is used
   - Use build mode to verify opencode's default build agent is used
   - Verify no Sisyphus-related messages appear

3. **Verify Sisyphus is disabled**:
   - Try to use `@Sisyphus` or `@orchestrator-sisyphus` (should show agent not available)
   - Check that agent selection menu doesn't show Sisyphus options

## Rollback

If issues occur, restore from chezmoi:
```bash
chezmoi apply
```

This will restore the original configuration file.
