# Verification Rules

## 検証レベル：標準

- テスト通過
- 自己レビュー
- 基本的な動作確認

## 検証フロー

### 実装後の検証（Buildエージェント）

1. **テスト実行**
   ```bash
   # 該当するテストコマンドを実行
   ```

2. **静的解析**
   ```bash
   # 型チェック、リントなど
   ```

3. **spec-reviewerによるレビュー**
   - 計画通りの実装か確認
   - 不足・過剰がないか確認

4. **code-reviewerによるレビュー**
   - コード品質確認
   - ベストプラクティス遵守確認

### 検証合格基準

| チェック | 基準 |
|---------|------|
| テスト | すべてPASS |
| 型チェック | エラー0 |
| spec-reviewer | 承認 |
| code-reviewer | 承認 |

## 不合格時の対応

1. 実装サブエージェントに修正依頼
2. 再検証
3. 合格するまで繰り返す

## 検証ツール

- `lsp`ツール：診断情報取得
- `bash`ツール：テスト・コマンド実行
- `read`ツール：コード確認
